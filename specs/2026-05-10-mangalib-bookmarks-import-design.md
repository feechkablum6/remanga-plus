# Импорт закладок из MangaLib в Remanga

**Дата:** 2026-05-10
**Статус:** design approved, ожидает плана реализации

## Цель

Дать пользователю расширения возможность за два клика перенести все свои закладки с `mangalib.me` (включая статус и прогресс по главам) в закладки на `remanga.org`. Готового решения «MangaLib → Remanga» в сообществе нет; ближайшие аналоги (`Tsun_Importer` для Atsu.moe, `MAL-Sync`, `manga-helper-standalone`) либо работают с другими целевыми сайтами, либо лишь сравнивают наличие манги между платформами.

## Точка входа

- Расширение Chrome (Manifest V3), уже установлено на `remanga.org`. Новой «второй» точки входа на mangalib не делаем — только popup расширения.
- В **popup'е** появляется секция «Импорт закладок из MangaLib».
- Сверху всегда виден **статус авторизации** для обоих сайтов:
  - `MangaLib`: `✓ Вошли как <имя>` или `✗ Не авторизован — Войти на mangalib.me` (ссылка открывает логин).
  - `Remanga`: `✓ Вошли как <имя>` или `✗ Не авторизован — Войти на remanga.org`.
  - Рядом — кнопка «Проверить ещё раз». Статус проверяется при каждом открытии popup'а.
- Кнопка **«Импорт закладок из MangaLib»** активна только если оба статуса зелёные.

## Основной сценарий

1. Клик «Импорт закладок из MangaLib» в popup'е → открывается **новая вкладка** со страницей расширения `import.html`. Popup можно закрывать.
2. На странице импорта:
   - Шапка со статусом авторизации обоих сайтов (тот же блок, что и в popup'е, чтобы было видно если пользователь разлогинился во время процесса).
   - Полоса прогресса «Получаю закладки с MangaLib… N / ~M».
   - Полоса прогресса «Ищу совпадения на Remanga… N / M».
3. После сбора и поиска — **превью** в виде таблицы:
   - ✅ зелёные строки — нашли уверенно (название почти совпало).
   - ⚠ жёлтые строки — спорно: можно выбрать из 2–3 кандидатов в выпадающем списке либо снять галочку.
   - ❌ серые без галочки — не нашли вовсе.
   - 🔁 серые с галочкой по умолчанию снятой — уже в закладках на Remanga (видно с пометкой «уже есть»).
   - Колонки: обложка/название MangaLib, статус MangaLib, последняя прочитанная глава MangaLib, кандидат на Remanga (имя + ссылка), статус совпадения, чекбокс «переносить».
   - Сверху таблицы — счётчики и фильтры по типу строк.
4. Кнопка **«Перенести отмеченные (N)»** → расширение последовательно (с rate-limit ~3–5 rps) добавляет тайтлы в закладки Remanga и отмечает прочитанные главы.
5. Финальный отчёт: «Перенесено: N. Пропущено уже существующих: M. Не найдено: K. Не удалось добавить: E (с причинами)». Кнопка «Скачать отчёт .txt» с полным списком, чтобы потом вручную добить несматченные.

**По «горячему пути» — два клика:** «Импорт закладок из MangaLib» (popup) → «Перенести отмеченные» (страница импорта).

## Правила обработки

### Поиск тайтла на Remanga

- Запрос к публичному поиску Remanga по названию из MangaLib (русское имя, при отсутствии — английское/ромадзи).
- Берём топ-3 кандидатов.
- Совпадение «уверенное», если нормализованная строка (lowercase, обрезка пробелов и пунктуации, приведение «ё/е») совпала точно. Иначе — «спорно».
- Если поиск пустой — «не найдено».

### Маппинг статусов

Дефолтный, без UI-настройки:

| MangaLib                | Remanga                |
|-------------------------|------------------------|
| Читаю                   | Читаю (0)              |
| В планах / Запланировано| Буду читать (1)        |
| Прочитано               | Прочитано (2)          |
| Брошено                 | Брошено (3)            |
| Отложено                | Отложено (4)           |
| Любимое                 | Читаю (0) (на Remanga такой категории нет) |

Точные имена статусов MangaLib подтверждаем при разведке API; маппинг хардкодится одной таблицей. Конкретные числовые коды Remanga — по `vraestoren/remanga.py` (0..5).

### Уже существует на Remanga

- Дубликаты определяются по найденному `title_id` Remanga: предварительно подгружаем закладки пользователя (`GET /api/users/<id>/bookmarks/?type=...` или эквивалент).
- В превью галочка снята по умолчанию, пометка «уже есть».
- Пользователь может вручную включить галочку и тогда мы делаем **повторный** `add_to_bookmarks` — поведение Remanga: либо обновит статус, либо вернёт ошибку, обрабатываем по факту. Перезапись прогресса в этом случае — да (просто отметим главы прочитанными, потеря прогресса с Remanga исключена, потому что отметка «прочитано» аддитивна).

### Прогресс по главам

- Из MangaLib берём номер последней прочитанной главы.
- На Remanga подтягиваем список глав найденного тайтла (берём дефолтную/основную ветку перевода).
- Отмечаем как прочитанные все главы с `chapter_number ≤ <последняя на mangalib>` через существующий механизм `markRemangaChapterAsViewed` (`POST /api/activity/views/`).
- Если на Remanga меньше глав — отмечаем что есть, остальное игнорируем без ошибки.

### Лимит скорости

- Запросы поиска: пакетами по 5–10 параллельно с ожиданием.
- Запросы записи (закладка + просмотры): последовательно с задержкой ~200 мс, целимся в 3–5 rps.
- При получении 429/5xx — экспоненциальный backoff, до 3 ретраев на запрос.

### Ошибки

- Ошибка на одном тайтле = пропустить, занести в «не удалось», продолжить.
- Сетевой обрыв / закрытие вкладки во время импорта: текущее состояние (что уже сделано, что осталось) сохраняется в `chrome.storage.local`. При следующем открытии popup'а — баннер «Прерванный импорт, продолжить или начать заново».

## Что НЕ делаем в первой версии

- Обратный импорт Remanga → MangaLib.
- Импорт оценок, лайков, тегов, личных коллекций.
- Ручной выбор ветки перевода на Remanga в превью (берём дефолтную).
- Поддержку родственных доменов: `yaoilib.me`, `hentailib.me`, `ranobelib.me`, `slashlib.me` и т. п. Делаем только `mangalib.me` / `mangalib.org`. Список доменов вынесен в константу — расширить позже легко, но из коробки только манга.
- Фоновый импорт после закрытия Chrome (закрыли браузер — процесс остановлен, при следующем запуске можно продолжить).
- Настраиваемый пользователем маппинг статусов.

## Архитектура (high-level)

Три зоны кода:

1. **Popup** (`src/popup.ts`, `public/popup.html`):
   - Блок статуса авторизации mangalib + remanga.
   - Кнопка «Импорт закладок из MangaLib» → `chrome.tabs.create({ url: chrome.runtime.getURL('import.html') })`.
   - Баннер «Прерванный импорт, продолжить» если в storage есть незавершённое состояние.

2. **Страница импорта** (новые файлы):
   - `public/import.html`, `src/import-page.ts` — UI таблицы превью, прогресса, отчёта.
   - `src/import-mangalib/fetch-bookmarks.ts` — клиент для забирания закладок из mangalib.me.
   - `src/import-mangalib/match-titles.ts` — нормализация названий и matching.
   - `src/import-mangalib/remanga-search.ts` — клиент поиска Remanga.
   - `src/import-mangalib/remanga-write.ts` — добавление закладок и отметка прогресса.
   - `src/import-mangalib/state.ts` — сериализация состояния в `chrome.storage.local` для resume.
   - `src/import-mangalib/status-mapping.ts` — таблица статусов.

3. **Service worker** (`src/background.ts`):
   - Хелпер `checkAuth(site)` — лёгкий запрос (`/api/me` или эквивалент) с user cookies.
   - Сообщения между popup ↔ страницей импорта (через storage events, не нужен прямой channel).

### Ключевые внешние API

| Эндпоинт                                                | Назначение                          |
|---------------------------------------------------------|-------------------------------------|
| `GET https://mangalib.me/api/bookmarks?...` (точный путь подтвердить разведкой) | Список закладок текущего пользователя со статусами и last-read chapter |
| `GET https://mangalib.me/api/auth/me` (или аналог)      | Проверка авторизации MangaLib       |
| `GET https://api.remanga.org/api/search/?query=...`     | Поиск тайтла Remanga по названию    |
| `GET https://api.remanga.org/api/users/current/`        | Проверка авторизации Remanga        |
| `GET https://api.remanga.org/api/users/<id>/bookmarks/` | Список существующих закладок        |
| `POST https://api.remanga.org/api/bookmarks/`           | Добавление закладки                 |
| `POST https://api.remanga.org/api/activity/views/`      | Отметка главы как прочитанной (уже используется в `premium-free.ts`) |

Точные пути и поля валидируем разведочными скриптами в `parser-server/scripts/` (паттерн `senkuro-*`, `inkstory-*`) перед тем, как зашить в код.

### Permissions (manifest.json)

Добавляются `host_permissions`:
- `https://*.mangalib.me/*`, `https://*.mangalib.org/*`
- `https://api.remanga.org/*`, `https://*.remanga.org/*` (часть уже есть)

Никаких дополнительных runtime-permissions от пользователя не требуется — куки используются прозрачно через `fetch({ credentials: 'include' })` из service worker / страницы расширения.

## Тесты (TDD)

Минимальный набор failing-тестов, которые пишутся первыми:

1. **Status mapping** (`tests/import-status-mapping.test.ts`)
   — каждая категория MangaLib → правильный код Remanga, неизвестная категория → дефолт «Читаю» с warning.
2. **Title matching** (`tests/import-title-matcher.test.ts`)
   — нормализация (регистр / пунктуация / ё/е / латиница).
   — точное совпадение → confidence «certain».
   — частичное (вхождение или Levenshtein > порога) → «ambiguous».
   — нет совпадений → «not_found».
3. **Bookmarks payload parser** (`tests/import-mangalib-bookmarks.test.ts`)
   — парсинг разведочных JSON-фикстур ответа MangaLib API → нормализованная структура `{ title, status, lastReadChapter, …}`.
4. **Chapter progress mapper** (`tests/import-chapter-progress.test.ts`)
   — список глав Remanga + последняя прочитанная mangalib → массив `chapter_id`, которые нужно отметить (включая случай «на Remanga меньше глав»).
5. **Resume state** (`tests/import-resume-state.test.ts`)
   — сериализация / десериализация `chrome.storage.local`, фильтрация уже выполненных шагов.
6. **End-to-end (моки)** (`tests/import-flow.test.ts`)
   — прогон полного цикла на моках: 5 закладок → 3 ✓, 1 ⚠, 1 ❌, дубликат пропускается, ошибка на одном продолжает работу.

Раннер — `node:test` + `node:assert/strict`, запуск через ad-hoc tsc + `node --test` (так же как остальные тесты в репо, см. CLAUDE.md проекта).

## Что нужно от пользователя для разработки

- Аккаунт на `mangalib.me` с реальными закладками: желательно несколько в разных статусах (Читаю, Прочитано с прогрессом, Брошено, В планах).
- Аккаунт на `remanga.org`, на котором не страшно добавить тестовые закладки (либо мы сделаем dev-флаг «dry-run», если пользователь захочет).
- Тестирование вживую — на машине пользователя через `npm run dev`, потому что ответы API зависят от региона / сессии.

## Риски и открытые вопросы

- **MangaLib API изменится** — `mangalib.me` исторически меняет домены и схему. Все вызовы изолируем за `import-mangalib/fetch-bookmarks.ts`, чтобы при поломке менять одно место.
- **Cloudflare / DDoS-Guard на mangalib** — если статически не пускает, fallback план: открывать `mangalib.me/bookmarks` в фоновой вкладке через `chrome.tabs.create({active: false})`, парсить через временный content script, закрывать вкладку. Решаем по факту разведки.
- **Маппинг номеров глав** между сайтами не всегда 1:1 (особенно при наличии extra-глав, prologue 0, или у webtoon с decimal-нумерацией). MVP-стратегия — сравнение по `chapter_number` с округлением до одного знака; что не сматчилось — игнорируем без ошибки.
- **Статус «Любимое» MangaLib** — возможно стоит маппить в «Читаю» только если ничего не уточнили; это поведение фиксируется тестом и при необходимости меняется одной строкой.
- **Бан / rate-limit Remanga** — если 429 будет регулярно, добавим экспоненциальный backoff и снизим параллелизм.
