## Установка на macOS

> Поддерживается Mac на Apple Silicon: M1, M2, M3, M4. Для Intel Mac готового установщика сейчас нет.

### 1. Установи `.pkg`

На [странице релизов](https://github.com/feechkablum6/remanga-plus/releases/latest) скачай **`Remanga-Plus.pkg`**.

Если macOS покажет «Не удалось открыть, разработчик не подтверждён», закрой окно, затем сделай **правый клик** по `.pkg` → **«Открыть»** → ещё раз **«Открыть»**. Дальше обычная установка: **«Продолжить» → «Установить»**.

После установки в **Программах** появится папка **Remanga Plus**. Внутри лежат parser-server, Node и папка расширения.

### 2. Загрузи расширение в браузер

Открой `chrome://extensions`, включи **«Режим разработчика»** и перетащи папку **`/Applications/Remanga Plus/extension`** в окно расширений.

<p align="center">
  <img src="screenshots/install-1-extensions-page.png" width="900" alt="Страница chrome://extensions с включённым режимом разработчика">
</p>

### 3. Открой remanga.org

Расширение и parser-server подключаются автоматически. В popup расширения статус должен быть **«Parser-server работает»**.

## Установка на Windows

1. Скачай **`Remanga-Plus-Setup.exe`** из последнего релиза в [Releases](https://github.com/feechkablum6/remanga-plus/releases/latest).
2. Запусти установщик. Если Windows покажет **«Windows protected your PC»**, нажми **«More info»** → **«Run anyway»**.
3. Пройди мастер установки: **Next** → **Install** → **Finish**. Установка идёт в `%LOCALAPPDATA%\Programs\Remanga Plus` без прав администратора.
4. В конце установщик предложит открыть setup helper. Он откроет папку `extension` и страницу расширений в Chrome или Edge.
5. На странице расширений включи **«Режим разработчика»**.
6. Нажми **«Загрузить распакованное»** и выбери открытую папку `extension`.
7. Открой [remanga.org](https://remanga.org).

После загрузки расширения parser-server запускается автоматически через Native Messaging. В меню Start появится папка **Remanga Plus** с ярлыками **Setup extension** и **Extension folder**.

### Удаление на Windows

Открой **Параметры** → **Приложения** → **Установленные приложения**, найди **Remanga Plus** и нажми **«Удалить»**. Расширение из браузера удаляется отдельно через `chrome://extensions`.

## Импорт закладок

В popup есть строка авторизации MangaLib и Remanga. Когда оба сайта авторизованы, кнопка **«Импорт →»** открывает страницу импорта.

Импорт проверяет существующие закладки на Remanga, подбирает тайтлы, переносит категории и прогресс глав. Если нужно авторизоваться, кликабельные названия сайтов в popup откроют нужные страницы в новой вкладке.

## Частые вопросы

**Premium Free не работает, что делать?**

1. Открой popup расширения и посмотри блок **«Сервис»**.
2. Если видишь, что parser-server не запущен, нажми кнопку перезапуска рядом со статусом.
3. Если статус зелёный/бирюзовый и указан порт `:7845` или соседний порт, сервер работает.
4. Если не помогло, переустанови актуальный `.pkg` или `.exe` из последнего релиза.

**Почему иногда появляется «Дальше главы нет»?**

Расширение сначала проверяет данные источника, затем пробует найти следующую главу по номеру. Если оба варианта не дали страниц, значит подходящую следующую главу сейчас найти не удалось.

**Можно ли использовать на Linux?**

Готового Linux-установщика сейчас нет. Чистка интерфейса может работать при ручной загрузке расширения, но Premium Free требует локальный parser-server и Native Messaging.

**Это безопасно?**

Код открыт. Premium Free работает через локальный parser-server на твоём компьютере. Расширение использует доступ к Remanga и MangaLib для чтения страниц, статуса авторизации, импорта закладок и проксирования запросов, которые браузер иначе блокирует.

**Почему расширения нет в Chrome Web Store?**

Premium Free вряд ли пройдёт ревью Chrome Web Store, поэтому установка пока ручная.

## Для разработчиков

Подробная архитектура, ограничения и ловушки описаны в [CLAUDE.md](CLAUDE.md).

```bash
npm install
npm run build       # собрать extension bundles в dist/
npm run check       # TypeScript: extension + native host
npm run dev         # watch только для content script
```

Parser-server — отдельный пакет:

```bash
cd parser-server
npm install
npm run dev
npm run check
npm test
```

Установщики:

```bash
npm run pkg:build          # macOS Apple Silicon .pkg
npm run pkg:test
npm run pkg:windows        # Windows x64 .exe, если локальный makensis рабочий
npm run pkg:windows:test
```

Корневого test runner в `package.json` нет. Тесты расширения запускаются вручную через `tsc` + `node --test`; точные команды и примеры есть в [CLAUDE.md](CLAUDE.md).
